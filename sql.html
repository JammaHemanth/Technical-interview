32.What is join and its types ?
It is used to combine records(rows) from 2 or more tables in an sql database based on a related columns between two tables.

  • Inner Join: It retrieves records that have matching values in both tables involved in the join that is widely used for join queries
    select city.id,city.countrycode,countrylanguage.countrycode,countrylanguage.language
    from city join countrylanguage
    on city.countrycode = countrylanguage.countrycode limit 10;

  • Left Join: It retrieves all records from the left and the matched records from right.
    select city.id,city.countrycode,countrylanguage.countrycode,countrylanguage.language
    from city left join countrylanguage
    on city.countrycode = countrylanguage.countrycode limit 10;

  • Right Join: It retrieves all records from the right and the matched records from left.
    select city.id,city.countrycode,countrylanguage.countrycode,countrylanguage.language
    from city right join countrylanguage
    on city.countrycode = countrylanguage.countrycode limit 10;

  • Full Join: It retrieves all records where there is a match either the left or the right table.
    select city.id, city.countrycode, countrylanguage.countrycode, countrylanguage.language
    from city full join countrylanguage
    on city.countrycode = countrylanguage.countrycode limit 10;

  • Self Join: It is a case of regular join where a table is joined itself based on some relation between it's own column.
    select c1.id, c1.countrycode, c2.id AS parent_id, c2.countrycode AS parent_countrycode
    from city AS c1
    inner join city AS c2
    on c1.parent_id = c2.id
    limit 10;

  • Cross join: i)It can be defined as an cartesian product of two tables involved in the join.
                     ii)It will combine each row from the first rowset with each row from the second rowset.
    select city.id, city.countrycode, countrylanguage.countrycode, countrylanguage.language
    from city
    cross join countrylanguage
    limit 10;

33.What are the types of errors in sql ?

  • Syntax error: It occurs when sql statement do not follow the correct synatx and structure of the language.
  • Semantic error: It occurs when sql statment is grametically correct but does not produce desired results due to incorrect logic.
  • Data type error: It occurs when data is inserted in a way that does not match the expected data type
  • Transaction error: These occurs when a transaction fails due to problem with locking
  • Constraint Validation: It occurs when the sql validates one or more constraint on the database
